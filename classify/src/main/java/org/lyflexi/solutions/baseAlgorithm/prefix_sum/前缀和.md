**计算前缀和: 左闭右开公式**, 定义前缀和`preS[idx]`为`元素和[0, idx)`, 则子数组`[left,right)` 的元素和为 `preS[right]−preS[left]`。由于是左闭右开, 因此`preS[0] == 0`

定义`preS[0] = 0`还有什么好处? 按照前缀和定义, 子数组`[0,right)` 的元素和为`preS[right]`, 此时也可以用`preS[right]−preS[0]`表示, 这统一了子数组元素和的计算

综上所述, `preS.length == nums.length + 1`

## 两个模板计算前缀和
见LC303.区域和检索 - 数组不可变

模板1: 遍历原数组`i:[0, n)`, `preS[i+1] = preS[i] + nums[i];`, 模板1更适合在遍历原数组`nums[]`的过程中迭代求解
```java
class NumArray {
    final int[] preS;
    public NumArray(int[] nums) {
        preS = new int[nums.length + 1];
        //遍历原数组索引
        for(int i = 0; i<nums.length; i++){
            preS[i+1] = preS[i] + nums[i];
        }
    }
    
    public int sumRange(int left, int right) {
        return preS[right+1] - preS[left];
    }
}
```

模板2: 直接遍历前缀和长度`i:[0, n+1)`, 此时计算`preS[i] = preS[i-1] + nums[i-1];`, 模板2更适合单独提前计算前缀和`preS[]`场景
```java
class NumArray {
    final int[] preS;
    public NumArray(int[] nums) {
        preS = new int[nums.length + 1];
        //遍历前缀和索引
        for(int i = 1; i<nums.length+1; i++){
            preS[i] = preS[i-1] + nums[i-1];
        }
    }
    
    public int sumRange(int left, int right) {
        return preS[right+1] - preS[left];
    }
}
```
上述模板1和模板2都互相进行代码风格转换, 但遇到具体问题选用最适合的模板
- 如适合模板1的题目: 见LC3427. 变长子数组求和
- 如适合模板2的题目: 有很多预处理计算场景都适合模板2

## 位运算前缀异或和
有一个正整数数组 `arr`，现给你一个对应的查询数组 queries，其中 `queries[i] = [Li, Ri]`。

对于每个查询 `i`，请你计算从 `Li` 到 `Ri` 的 `XOR` 值, 即 `arr[Li] xor arr[Li+1] xor ... xor arr[Ri]`作为本次查询的结果。

并返回一个包含给定查询 queries 所有结果的数组。

见LC1310. 子数组异或查询, 答案是两个异或前缀和再做异或即为差值
```java
    public int[] xorQueries(int[] arr, int[][] queries) {
        int[] preS = new int[arr.length+1];
        int[] ans = new int[queries.length];

        //1. 遍历原数组索引
        for(int i = 0; i<arr.length; i++){
            preS[i+1] = preS[i]^arr[i];
        }
        //2. 遍历前缀和索引
//        for(int i = 1; i<arr.length+1; i++){
//            preS[i] = preS[i-1]^arr[i-1];
//        }

        int idx = 0;
        for(int[] query: queries){
            ans[idx++] = preS[query[1] + 1]^preS[query[0]];
        }

        return ans;
    }
```

## 预处理数组前缀和
见LC3152. 特殊数组 II, 求真正的前缀和之前, 先要将长度为`n`的`nums`数组转换为`n-1`对奇偶对判断数组`a[]`
```java
    public boolean[] isArraySpecial(int[] nums, int[][] queries) {
        int n = nums.length, newLen = n-1;
        //预处理数组, 将长度为n的nums数组转换为n-1对奇偶对判断
        int[] a = new int[newLen];
        for(int i = 0; i<n-1;i++){
            //第二个^1操作是对两数异或后末尾二进制位取反
            //最后的&1操作是用来对上述结果求模2
            a[i] = (nums[i+1] ^ nums[i] ^1) & 1;
        }

        //左闭右开公式
        //1. 遍历原数组索引
        int[] preS = new int[newLen+1];
        for(int i = 0; i<newLen; i++){
            preS[i+1] = preS[i] + a[i];
        }
        //2. 遍历前缀和索引
//        for(int i = 1; i<newLen+1; i++){
//            preS[i] = preS[i-1] + a[i-1];
//        }

        boolean[] ans = new boolean[queries.length];
        int idx = 0;
        for(int[] query: queries){
            //对应区间query长度-1个元素a[i], a[]的区间和恰好是preS[query[1]] - preS[query[0]], query[1]不用再加1了
            //如nums[x1,x2,x3], 对应于a[y1,y2]
            ans[idx++] = (preS[query[1]] - preS[query[0]])>0? false: true;
        }

        return ans;
    }
```

## 贪心计算前缀和
见53.最大子数组和, 这类题目贪心求解的思想基础是121.买卖股票的最佳时机

或者见更复杂点的 1749.任意子数组和的绝对值的最大值

- 在计算`subMxs`的过程中, 被减数贪心的寻找最小前缀和`mnPreS`, 迭代`subMxs`和`mnPreS`
- 在计算`subMns`的过程中, 被减数贪心的寻找最大前缀和`mxPreS`, 迭代`subMns`和`mxPreS`
```java
    public int maxAbsoluteSum(int[] nums) {
        //迭代计算
        int n = nums.length;
        //求最大子数组和
        int subMxs = Integer.MIN_VALUE, mnPreS = 0;
        //求最小子数组和
        int subMns = Integer.MAX_VALUE, mxPreS = 0;
        int[] preS = new int[n+1];
        for(int i = 0; i<n; i++){
            preS[i+1] = preS[i] + nums[i];
            subMxs = Math.max(subMxs, preS[i+1] - mnPreS);
            subMns = Math.min(subMns, preS[i+1] - mxPreS);

            mnPreS = Math.min(mnPreS, preS[i+1]);
            mxPreS = Math.max(mxPreS, preS[i+1]);
        }

        return Math.max(subMxs, -subMns);
    }
```

在此类贪心题目中, 我们不必计算出整个前缀和数组`preS[]`, 这可以进行空间优化
```java
    public int maxAbsoluteSum(int[] nums) {
        //迭代计算
        int n = nums.length;
        //求最大子数组和
        int subMxs = Integer.MIN_VALUE, mnPreS = 0;
        //求最小子数组和
        int subMns = Integer.MAX_VALUE, mxPreS = 0;
        int preS = 0;//前缀和空间优化
        for(int i = 0; i<n; i++){
            preS += nums[i];
            subMxs = Math.max(subMxs, preS - mnPreS);
            subMns = Math.min(subMns, preS - mxPreS);

            mnPreS = Math.min(mnPreS, preS);
            mxPreS = Math.max(mxPreS, preS);
        }
        return Math.max(subMxs, -subMns);
    }
```

## 双前缀和问题
见LC3652.按策略买卖股票的最佳时机

```java
    public long maxProfit(int[] prices, int[] strategy, int k) {
        int n = prices.length;

        long[] preS = new long[n+1];//前缀和: prices[i] * strategy[i]
        long[] preSNew = new long[n+1];//前缀和:prices[i]

        for(int i = 1; i< n+1; i++){
            preS[i] = preS[i-1] + prices[i-1] * strategy[i-1];
            preSNew[i] = preSNew[i-1] + prices[i-1];
        }

        //init ans, before modify strategy
        long ans = preS[n];

        //枚举窗口的右端点, 只有当枚举到第k-1个元素的时候, 才可以容纳修改策略, 恰好从0开始[0, k-1]个
        for(int i = k; i<=n; i++){
            //每当修改策略的时候, 划分数组为四分
            //A: preS[i-k];
            //B: 0
            //C: preSNew[i] - preSNew[i-1-k/2+1]
            //D: preS[n] - preS[i]
            ans = Math.max(ans, preS[i-k] + preSNew[i] - preSNew[i - k/2] + preS[n] - preS[i]);
        }

        return ans;
    }
```

## 前缀和与哈希表
我们需要统计和为`k`的子数组, 基于前缀和求解`k = s2 - s1`, 类似于两数之和问题`a1 + a2 = k`, 我们可以用哈希做辅助结构

如LC560.和为 K 的子数组, 关键点在于, 我们需要初始化空前缀和的次数为`1`, 即`map.put(0, 1)`以满足满足`s2`本身就是答案的情况

这是子数组计算不同于两数之和的地方, 因为两数之和必须是两个数与目标`k`, 而子数组存在前缀和`[0, r+1)`本身就是子数组且满足目标`k`的情况
```java
    public int subarraySum(int[] nums, int k) {
        //前缀和, 并且因为是两个变量用哈希
        int n = nums.length, ans = 0;
        int[] preS = new int[n+1];
        Map<Integer, Integer> map = new HashMap<>();
        map.put(0, 1);//空前缀和的次数

        //下面循环了n次,  少循环一次就是对(preS[0]=0, 1)的map统计, 因为我们在循环外提前初始化了map.put(0, 1)
        for(int r = 0; r< n; r++){
            //不包含
            preS[r+1] = preS[r] + nums[r];
            //s2 - s1 = k
            ans += map.getOrDefault(preS[r+1] - k, 0); 
            map.merge(preS[r+1], 1 ,Integer::sum);
        }
        return ans;
    }
```

更好理解的方式是用两次循环, 第一次循环单独计算前缀和数组`preS[]`
```java
    public int subarraySum(int[] nums, int k) {
        //前缀和, 并且因为是两个变量用哈希
        int n = nums.length, ans = 0;
        int[] preS = new int[n+1];
        for(int i = 1; i< n+1; i++){
            preS[i] = preS[i-1] + nums[i-1];
        }

        Map<Integer, Integer> map = new HashMap<>();
        for(int i = 0; i< n+1; i++){
            //s2 - s1 = k
            ans += map.getOrDefault(preS[i] - k, 0); 
            //会初始化preS[0]的次数为1, 这很关键, 能够满足s2本身就是答案的情况
            map.merge(preS[i], 1 ,Integer::sum);
        }

        return ans;
    }
```
