**计算前缀和: 左闭右开公式**, 定义前缀和`preS[idx]`为`元素和[0, idx)`, 则子数组`[left,right)` 的元素和为 `preS[right]−preS[left]`。由于是左闭右开, 因此`preS[0] == 0`

定义`preS[0] = 0`还有什么好处? 按照前缀和定义, 子数组`[0,right)` 的元素和为`preS[right]`, 此时也可以用`preS[right]−preS[0]`表示, 这统一了子数组元素和的计算

综上所述, `preS.length == nums.length + 1`

## 双前缀和问题

    public long maxProfit(int[] prices, int[] strategy, int k) {
        int n = prices.length;

        long[] preS = new long[n+1];//前缀和: prices[i] * strategy[i]
        long[] preSNew = new long[n+1];//前缀和:prices[i]

            preSNew[i] = preSNew[i-1] + prices[i-1];
        }

        //init ans, before modify strategy
        long ans = preS[n];

        //枚举窗口的右端点, 只有当枚举到第k-1个元素的时候, 才可以容纳修改策略, 恰好从0开始[0, k-1]个
        for(int i = k; i<=n; i++){
            //每当修改策略的时候, 划分数组为四分
            //A: preS[i-k];
            //B: 0
            //C: preSNew[i] - preSNew[i-1-k/2+1]
            //D: preS[n] - preS[i]
            ans = Math.max(ans, preS[i-k] + preSNew[i] - preSNew[i - k/2] + preS[n] - preS[i]);
        }

    }

## 前缀和空间优化
见53.最大子数组和, 这类题目贪心求解的思想基础是121.买卖股票的最佳时机

或者见更复杂点的 1749.任意子数组和的绝对值的最大值

- 在计算`subMxs`的过程中, 被减数贪心的寻找最小前缀和`mnPreS`, 迭代`subMxs`和`mnPreS`
- 在计算`subMns`的过程中, 被减数贪心的寻找最大前缀和`mxPreS`, 迭代`subMns`和`mxPreS`
```java
    public int maxAbsoluteSum(int[] nums) {
        //迭代计算
        int n = nums.length;
        //求最大子数组和
        int subMxs = Integer.MIN_VALUE, mnPreS = 0;
        //求最小子数组和
        int subMns = Integer.MAX_VALUE, mxPreS = 0;
        int[] preS = new int[n+1];
        for(int i = 0; i<n; i++){
            preS[i+1] = preS[i] + nums[i];
            subMxs = Math.max(subMxs, preS[i+1] - mnPreS);
            subMns = Math.min(subMns, preS[i+1] - mxPreS);

            mnPreS = Math.min(mnPreS, preS[i+1]);
            mxPreS = Math.max(mxPreS, preS[i+1]);
        }

        return Math.max(subMxs, -subMns);
    }
```

我们不必计算出整个前缀和数组`preS[]`, 通过滚动计算前缀和, 同时去除了额外的空间复杂度以及单独构造前缀和数组的额外循环
```java
    public int maxAbsoluteSum(int[] nums) {
        //迭代计算
        int n = nums.length;
        //求最大子数组和
        int subMxs = Integer.MIN_VALUE, mnPreS = 0;
        //求最小子数组和
        int subMns = Integer.MAX_VALUE, mxPreS = 0;
        int preS = 0;//前缀和空间优化
        for(int i = 0; i<n; i++){
            preS += nums[i];
            subMxs = Math.max(subMxs, preS - mnPreS);
            subMns = Math.min(subMns, preS - mxPreS);

            mnPreS = Math.min(mnPreS, preS);
            mxPreS = Math.max(mxPreS, preS);
        }
        return Math.max(subMxs, -subMns);
    }
```

## 前缀和与哈希表
我们需要统计和为`k`的子数组, 基于前缀和求解`k = s2 - s1`, 类似于两数之和问题`a1 + a2 = k`, 我们可以用哈希做辅助结构

如LC560.和为 K 的子数组, 关键点在于, 我们需要初始化空前缀和的次数为`1`, 即`map.put(0, 1)`以满足满足`s2`本身就是答案的情况

这是子数组计算不同于两数之和的地方, 因为两数之和必须是两个数与目标`k`, 而子数组存在前缀和`[0, r+1)`本身就是子数组且满足目标`k`的情况
```java
    public int subarraySum(int[] nums, int k) {
        //前缀和, 并且因为是两个变量用哈希
        int n = nums.length, ans = 0;
        int preS = 0;//空间优化
        Map<Integer, Integer> map = new HashMap<>();
        map.put(0, 1);//空前缀和的次数
    
        //下面循环了n次,  少循环一次就是对preS[0]=0的map统计, 因为初始化map已经补充了map.put(0, 1)而不是map.put(0, 0)
        for(int r = 0; r< n; r++){
            //不包含
            preS += nums[r];
            //s2 - s1 = k
            ans += map.getOrDefault(preS - k, 0);
            map.merge(preS, 1 ,Integer::sum);
        }
        return ans;
    }
```

更好理解的方式是用两次循环, 第一次循环单独计算前缀和数组`preS[]`
```java
    public int subarraySum(int[] nums, int k) {
        //前缀和, 并且因为是两个变量用哈希
        int n = nums.length, ans = 0;
        int[] preS = new int[n+1];
        for(int i = 1; i< n+1; i++){
            preS[i] = preS[i-1] + nums[i-1];
        }

        Map<Integer, Integer> map = new HashMap<>();
        for(int i = 0; i< n+1; i++){
            //s2 - s1 = k
            ans += map.getOrDefault(preS[i] - k, 0); 
            //会初始化preS[0]的次数为1, 这很关键, 能够满足s2本身就是答案的情况
            map.merge(preS[i], 1 ,Integer::sum);
        }

        return ans;
    }
```
总结就是, 如果我们求解目标是个数, 那么哈希里面存count

基于LC560前缀和与哈希的变形题
1. 改成计算元素和等于 k 的最短子数组，要怎么做？
答：求最短，哈希表的value里不再存出现的个数，应该存最后一次出现的下标，这样计算出来的长度最短。
2. 改成计算元素和等于 k 的最长子数组，要怎么做？
答：求最长，哈希表的value里不再存出现的个数，应该存第一次出现的下标，这样计算出来的长度最长。
3. 改成计算元素和等于 k 的所有子数组的长度之和，要怎么做？
答：求长度之和，需要两个哈希, 一个哈希里面存个数count, 一个哈希里面存下标和sumIdx, 答案累加ans += count * i - sumIdx;
4. 改成元素和至多为 k，要怎么做？
答：哈希表变成有序的红黑树，treemap，这样key从匹配s-k到能匹配>=s - k的所有方案.
5. 改成计算元素和为奇数的子数组个数，要怎么做？
答：哈希表的key不存具体某个s的个数，哈希的长度为2仅存奇和偶的个数，遇到偶加奇数个, 遇到奇加偶数个

列成表如下:
- 求个数, map里面存count
- 求最短, map里面存最新下标
- 求最长, map里面存最早下标
- 求长度和, map里面存`int[2]`, 即`int[0]`存`count`, `int[1]`存下标和, 或者用两个map
- 求≤ k, 用有序map红黑数
- 求奇偶, map里面存2种状态

## pair前缀和
_`HashMap`的`key`是一对值`pair`, 并且这对值`pair`是与当前`value`捆绑的, 因此我们不能将其视为双前缀和问题_

在 `Java` 中, 此类问题需要注意`map.containsKey`引用判断失效的问题.

默认情况下使用 `containsKey` 方法并以引用对象作为键时，判断是否"准确"取决于该对象类是否正确重写了 `hashCode()` 和 `equals()` 方法。

简单来说，HashMap 的查找机制分为两步： 
- 定位桶（Bucket）：首先调用键对象的 hashCode() 方法，计算出哈希值，以此确定该键值对在哈希表中的存储位置（即“桶”）。
- 判断相等：在定位到的桶中，遍历所有键值对，并调用键对象的 equals() 方法与目标键进行逐一比较，以确认是否找到了完全相同的键。

如果你的自定义类没有重写 hashCode() 和 equals()方法，它们会沿用 Object 类的默认实现：
- hashCode()：通常基于对象的内存地址计算。
- equals()：比较两个对象的内存地址（即使用 == 操作符）。

在这种情况下，`containsKey` 的判断是基于对象引用的。即使你创建了两个属性值完全相同的对象，它们在内存中也是两个不同的实例，拥有不同的内存地址。因此，`containsKey` 会返回 `false`，这通常不符合我们的业务逻辑预期。

为了让 HashMap 根据对象的内容来判断键是否存在，你必须在自定义类中同时重写 hashCode() 和 equals() 方法。 

重写时需要遵循一个核心契约： 如果两个对象通过 equals() 方法判断为相等，那么它们的 hashCode() 方法必须返回相同的值。

除此之外, 我们还要保证字段的不可变性, 一旦对象被放入 HashMap，它就会被存储在由其 hashCode() 决定的特定“桶”中。如果你随后修改了该对象的字段，导致其 hashCode() 值发生变化，那么当你再次使用这个修改过的对象调用 containsKey 或 get 时，HashMap 会去一个新的、错误的“桶”里查找，结果自然是找不到。

我们可以直接使用 Java 16+ 引入的 record 创建一个类，自动生成 final 字段、构造方法、hashCode、equals 和 toString，代码更简洁、可读性更高：
```java
record Pair(int preS, int xorS){}
```

见LC3755. 最大平衡异或子数组的长度
```java
    //record自动生成 final 字段、构造方法、hashCode、equals 和 toString
    record Pair(int preS, int xorS){}
    public int maxBalancedSubarray(int[] nums) {
        //把 0 看成 −1，计算和为 0 的最长子数组。这样就有了子数组和的比较对象target == 0
        int n = nums.length;
        int ans = 0, preS = 0,  xorS = 0; //两个前缀和数组都进行空间优化
        //存储最早出现的下标, key是二元组[preS, xorS]
        Map<Pair, Integer> map = new HashMap<>();
        map.put(new Pair(0, 0), -1);//适配前缀和本身就是子数组的情况
        for(int r = 0; r< n; r++){
            //s2 - s1 == 0, 即s2 == s1, 求len(s2 -s1)
            preS += (nums[r] & 1)==0?1:-1;
            xorS ^= nums[r];
            Pair pair = new Pair(preS, xorS);
            if(map.containsKey(pair)){
                ans = Math.max(ans, r - map.get(pair));
            }else{
                //store earliest idx
                map.put(pair, r);
            }

        }

        return ans;
    }
```
