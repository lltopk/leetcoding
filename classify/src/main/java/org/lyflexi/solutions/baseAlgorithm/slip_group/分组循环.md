## 分组循环
适用场景：按照题目要求，数组会被分割成若干组，且每一组的判断/处理逻辑是一样的。

核心思想：
- 外层循环负责遍历组之前的准备工作（记录开始位置），和遍历组之后的统计工作（更新答案最大值）。
- 内层循环负责遍历组，找出这一组最远在哪结束。

这个写法的好处是，各个逻辑块分工明确，也不需要特判最后一组（易错点）。以我的经验，这个写法是所有写法中最不容易出 bug 的，推荐大家记住。
```java
n = len(nums)
i = 0
while i < n: //或者 for (int i = 0; i < n; )
    start = i
    while i < n and ...:
        i += 1 
    # 从 start 到 i-1 是一组, 长度可以直接表示为i - start
    # 下一组直接从 i 开始，无需外循环末尾 i += 1
```

时间复杂度乍一看是 O(n^2)，进行摊还分析发现变量 i 只会增加，不会重置也不会减少。最内层的`i += 1`至多执行N次, 所以二重循环总共循环 O(n) 次，所以时间复杂度是 O(n)。

见LC2760. 最长奇偶子数组
```java
/**
 * 分组循环
 */
public class LC2760_longestAlternatingSubarray {
    public int longestAlternatingSubarray(int[] nums, int threshold) {
        int n = nums.length;
        int ans = 0, i = 0;
        while (i < n) {
            if(nums[i] > threshold || nums[i] % 2 != 0){
                i++;
                continue;
            }
            int start = i++; // 记录这一组的开始位置已经满足条件,  并且从i+1开始新的计算
            while (i < n && nums[i] <= threshold && nums[i] % 2 != nums[i - 1] % 2) {
                i++;
            }
            // 从 start 到 i-1 是满足题目要求的（并且无法再延长的）子数组
            ans = Math.max(ans, i - start);
        }
        return ans;
    }
}
```
