## 滑动窗口
在滑动窗口计算过程中, 我们习惯于**枚举窗口右端点`nums[r] r==i`**, 有以下两个数学规律
- 此时窗口内连续子数组个数`[l,r], [l+1,r]...[r,r]`之和恰好等于窗口长度`r-l+1`
- 此时窗口左侧连续子数组个数`[l-1,r], [l-2,r]...[0,r]`之和恰好等于`l`, 其中`l-1`不在窗口中

复杂度分析:
- 不要在内循环中遍历窗口计算, 这将会是O(nk)复杂度, 其中k为窗口大小. 滑动窗口的本质是`O(1)`计算入`compute(nums[r]){...}`以及计算出`compute(nums[l]){...l++;}`
- `O(n*1)`分析: 由于内层循环`l`最大加到`n`, 平均每轮外循环加`1`, 因此最终时间复杂度为`O(n*1)`
- `O(n)`分析: 由于内层循环`l`只增不减, **以二重循环n*n为例, 乘法的本质是计算最内层循环执行次数`O(n^2)`**, 但对于滑动窗口直观看出`O(n)`

在正式枚举窗口右端点之前, 需要定义以下变量:
- helper, 辅助变量或者辅助数组或者辅助哈希, 动态存储当前窗口下的计算状态
- ans, 迭代最终答案
- l, 窗口左指针

在枚举窗口右端点的过程中, 要记得窗口内计算状态的变化是由枚举窗口右端点引起的, 这一点很重要

如果给出字符串, 建议提前转为字符数组`char[] arr = s.toCharArray()`

### 定长窗口模板
以LC643为例, 子数组最大平均数 I
- 计算入r: 对每个枚举值(窗口右端点)计算
- 计算出l: 当窗口达到固定长度, 迭代结果`ans = Math.?(ans, helper/k)`, 同时计算并收缩左指针`{helper-=nums[l]; l++;}`
```java
    public double findMaxAverage(int[] nums, int k) {
        double ans = -10000.0;//求最大
        double helper = 0;
        int l = 0;
        //定长滑动窗口
        for(int r = 0; r<nums.length; r++){
            helper+=nums[r];
            //一直枚举右端点, 等到窗口形成
            if(r-l+1 == k){
                ans = Math.max(ans, helper/k);
                helper -=nums[l];
                l++;
            }
        }
        return ans;
    }
```

### 不定长窗口求长度, shrink求最小
以LC209为例, 长度最小的子数组
- 外层循环枚举r: 对每个枚举值(窗口右端点)计算
- while循环收缩l: **确定窗口收缩条件进而移动左指针**, 复杂案例可以对该关键条件抽象为收缩shrink函数
- while循环结束迭代解: 因为是求最小，所以最后需要加额外的特判条件(满足题意的)迭代解，否则会求解错误
```java
    public int minSubArrayLen(int target, int[] nums) {
        //不定长滑动窗口求最小， 越长越合法
        int n = nums.length;
        int ans = n+1;//求最小
        int l = 0;
        int helper = 0;
        for(int r  =0; r<n; r++){
            helper+=nums[r];
    
            //窗口收缩条件， 如果用小于那么窗口无法收缩
            while(helper - nums[l] >= target){//编程技巧： 因为循环条件为真则收缩窗口，这里提前减去helper-nums[l]目的是省去最后一次循环，避免最终窗口左边界l多1
                helper-=nums[l];
                l++;
            }
    
            // 因为是求最小，所以最后需要加额外的判断当满足题意helper >= target迭代解， 否则会求解错误
            if(helper >= target){
                ans = Math.min(ans, r - l + 1);//while循环外迭代迭代答案
            }
        }
    
        return ans==n+1?0:ans;
    }
```

### 不定长窗口求长度, shrink求最大
以LC424为例, 替换后的最长重复字符

_模板与求最小类似, 唯一的区别是解迭代计算Math.max, 正因为是求最大因此可以省去特判条件_
```java
    public int characterReplacement(String s, int k) {
        //不定长滑动窗口求最大
        char[] arr = s.toCharArray();
        int n = arr.length;
        int ans = 0, l = 0;
        int[] dict = new int[26];
        for(int r = 0; r<n ;r++){
            dict[arr[r] - 'A']++;
            //窗口收缩条件
            // 窗口所有字符出现总次数 - 窗口中频率最高字符次数 = other其他字符出现的次数和 > k
            // 这是贪心的思想, 也就是我们优先替换频率低的, 而不去替换频率高的
            while(shrink(dict, k)){
                dict[arr[l] - 'A']--;
                l++;
            }
    
            ans = Math.max(ans, r - l +1 );
        }
    
        return ans;
    }
    
    private boolean shrink(int[] dict , int k ){
        int sum = 0;
        int max = 0;
        for(int i = 0; i< 26 ;i++){
            sum+=dict[i];
            max = Math.max(max, dict[i]);
        }
    
        return sum - max > k;
    } 
```

### 不定长窗口求个数, shrink越短越合法
以LC713为例, 乘积小于 K 的子数组

模板与求长度类似, 唯一的区别是解迭代求的是窗口个数

_由于是越短越合法, 因此窗口长度`l, l+1...r`即`r - l + 1`恰好为题意个数, 也可以省去特判条件_
```java
    public int numSubarrayProductLessThanK(int[] nums, int k) {
        //1 <= nums[i] <= 1000
        if(k<=1){
            return 0;
        }
        int helper = 1;
        int l= 0;
        int cnt = 0;
        //滑动窗口, 以右端点为终点
        for(int r = 0; r<nums.length; r++){
            helper*=nums[r];
            while(helper>=k){
                helper/=nums[l];
                l++;
            }
            //右端点为终点, 当乘积严格小于k的时候, 连续子数组的个数为当前序列长度
            cnt+=r-l+1;
        }

        return cnt;
    }
```

### 不定长窗口求个数, shrink越长越合法
LC1358, 包含所有三种字符的子字符串数目

模板与求长度类似, 唯一的区别是解迭代求的是窗口个数

_由于是越长越合法, 窗口左侧长度`0...l-1`共计`l`恰好为题意个数, 也可以省去特判条件_
```java
    public int numberOfSubstrings(String s) {
        //辅助数组, 代表子串a,b,c字符的个数
        int[] helper = new int[3];
        int ans = 0;
        int l = 0;
        for(int r = 0; r<s.length(); r++){
            helper[s.charAt(r) - 'a']++;
            //滑动窗口求个数, 越长越合法
            while(helper[0]>0 && helper[1]>0 &&helper[2]>0){
                helper[s.charAt(l) - 'a']--;
                l++;
            }
            ans += l;
        }

        return ans;
    }
```

### 不定长滑动窗口shrink函数O(1)优化
在不定长滑动窗口模板中, 最关键的是明确窗口收缩条件`while(shrink())`

_对于有些困难题来说, 有的时候shrink复杂度过高, 一般是`O(k)`, 其中`k`是字典数组的长度, 那有没有进一步的优化呢?_

那就是引入新的计数器`counter`, 根据题意赋予`counter`具体的含义, 此时窗口收缩条件就等价与`while(counter == target)`

此类题目有
- LC76 最小覆盖子串
- LC632 最小区间
