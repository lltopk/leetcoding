
## 栈ArrayDeque
Java中的`Stack`是线程同步工具类, 每个方法都有`synchronized`声明, 并且底层是`Vector`模型, 比较古老, 在大数据量频繁操作的场景下(leetcode)性能会很差

`leetcode`场景下推荐使用`Deque<Integer> stack = new ArrayDeque<>();`, 是专门为栈和队列设计的, 同时支持队列和栈, 性能好
```text
入队: offer(e)->addLast(e)
出队: remove()->removeFirst()

入栈: push(e)->addFirst(e)
出栈: pop()->removeFirst()
```

## 列表/数组/字符串模拟栈
用列表/数组/字符串模拟栈的好处还有就是能够按照自底向上的扫描顺序还原字符串, 见71. 简化路径
```java
    public String simplifyPath(String path) {
        List<String> stack = new ArrayList<>();
        for(String s: path.split("/")){
            if("".equals(s) || ".".equals(s)){
                continue;
            }
            if(!"..".equals(s)){
                stack.add(s);
            }else if(!stack.isEmpty()){
                stack.remove(stack.size()-1);
            }
        }
        return "/" + String.join("/", stack);
    }
```

下面给出每种模拟的实现例题
### 列表模拟
单独创建个列表/数组模拟栈，这样就不局限于栈提供的有限API, 通过数组索引支持随机访问这样性能更高， 见LC1472. 设计浏览器历史记录

与JDK保持一致的是, 我们列表/数组模拟栈的时候也习惯于将栈索引始终保持peek+1的位置, 因此实际栈顶元素是`nums[tail-1]`

如果题目明确栈大小, 你可以用数组模拟, 如果题目不明确栈大小, 你只能用列表模拟
```java
public class LC1472_BrowserHistory2 {
    //用列表/数组模拟栈， 支持随机访问性能更高
    final List<String> history = new ArrayList<>();
    //游标卡尺, tail指向peek+1位置, 因此本题索引范围是[0, tail-1]
    int tail = 0;//和jdk源码一样, tail永远指向实际位置的下个位置
    public LC1472_BrowserHistory2(String homepage) {
        history.add(homepage);
        tail++;
    }

    public void visit(String url) {
        //SubList内部持有的仍然是完整list的引用
        history.subList(tail, history.size()).clear();
        history.add(url);
        tail++;
    }

    public String back(int steps) {
        int cur = Math.max(0, tail-1-steps);
        tail = cur+1;
        return history.get(cur);
    }

    public String forward(int steps) {
        int cur = Math.min(history.size()-1, tail - 1 +steps);
        tail = cur + 1;
        return history.get(cur);
    }
}
```

### 数组模拟
需要注意的是当前下标`i`指的是模拟栈的`tail`索引, `nums[tail] === null`, 而不是题目输入数组的索引, 为了加深对列表/数组模拟栈的理解下面见LC946. 验证栈序列
```java
    public boolean validateStackSequences(int[] pushed, int[] popped) {
        int i = 0, j = 0, n = pushed.length;
        int[] helper = new int[n];
        //i是新数组helper的索引
        for(int num: pushed){
            //模拟栈, 这道题栈索引i始终保持peek+1的位置
            helper[i++] = num;
            while(i>0 && helper[i-1] == popped[j]){
                i--;
                j++;
            }
        }
        return i == 0;
    }
```


### 字符串模拟
`String.join("delimiter", "listRef")`按照相对顺序返回字符串答案的前提是, `listRef`中元素类型必须是`CharSequence`如`String`, 而不能是`Character`

很多题目的输入都是`char[] sa = s.toCharArray();`, 因此有必要单独对`Character`类型的元素题目单独讨论, 我们可以用`StringBuilder`处理这类问题

`StringBuilder`提供了类似于`List`的根据索引随机访问字符的`API`, 见LC1047. 删除字符串中的所有相邻重复项
```java
    public String removeDuplicates(String s) {
        //返回原顺序的字符串, 因此用列表模拟栈最方便了
        StringBuilder stack = new StringBuilder();
        char[] sa = s.toCharArray();
        for(char c: sa){
            if(!stack.isEmpty() && c == stack.charAt(stack.length()-1)){
                stack.deleteCharAt(stack.length()-1);
                continue;
            }

            stack.append(c);
        }
        return stack.toString();
    }
```
