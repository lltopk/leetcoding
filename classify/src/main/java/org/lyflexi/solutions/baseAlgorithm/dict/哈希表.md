哈希常见尽量用int[]数组足以, 因为数组的本质也是字典, 并支持随机访问


## 整形数组int[]
对于字符场景, 我们知道ascii集总共有128个, 十进制从0~127
- 最小字符是NULL, 不可见, 但可以表示为'\0'
- 最大字符是DEL, 不可见, 无法表示

其中可见字符位于ascii表中间, 包括英文字母、数字、符号和空格, 这共有95个, 十进制从32~126
- 最小字符表示是'space'
- 'A'十进制是65
- 'a'十进制是97
- 最大值是'~'十进制为126

代码实现如下
```java
//字符char为整个ascii集, 共计128个, 将char - '\0'作为索引
int[] dict = new int[128];
//字符char为整个可见字符集, 共计95个, 将char - '空格'作为索引
int[] dict = new int[95];
//字符char为小写a~z, 共计26个, 将char - 'a'作为索引
int[] dict = new int[26];
```

## HashMap
Java的HashMap, 绝大部分应用场景, Key为字符
- V视作计数Count
- V视作集合ArrayList

### 数值V默认值计算
Map<Integer, Integer> helper = new HashMap<>();
```java
if(helper.get(arr[i]) == null){
    helper.put(arr[i], 1);
}else{
    helper.put(arr[i], helper.get(arr[i])+ 1);
}
```

getOrDefault函数简化
```java
helper.put(arr[i], helper.getOrDefault(arr[i], 0) + 1);
```

merge函数传入加法接口:
```java
//map.merge, 当key不存在则初始化value为newV, 当key存在则oldV+newV
// 其中第三个参数是BiFunction remappingFunction
helper.merge(arr[i], 1, (a,b)-> {return a+b;});
//or
helper.merge(arr[i], 1, Integer::sum);
```

### 集合V默认值计算
Map<Integer, List<Integer>> helper = new HashMap<>();

由于集合V执行add返回布尔值而不是集合本身, 因此getOrDefault只能初始化集合V, 无法直接给集合V追加元素

所以集合V场景下使用computeIfAbsent完成简化操作
```java
//computeIfAbsent返回对象V
helper.computeIfAbsent(1, (key)-> {return new ArrayList<>();}).add(1);
```
