链表中无非是三种指针： `pre`， `cur`， `next`, 然后是区分下面两种节点赋值操作：
- 给`p`赋值， 只是移动指针，不会修改链表
- 给`p.next`赋值才是修改链表, 指的是修改节点之间的连接关系，会改变链表结构

由于力扣周赛的链表题可以转成数组处理，这种投机做法导致难度比直接处理链表要低的多，事实上链表题目是很考验思维能力的。因此请带着如下的问题与思考去刷题：

一、什么时候需要构造新的链表`ListNode dummy = new ListNode(0, head)`? 
- 需要构造`dummy`: 便于统一处理特殊节点(头节点), 最终返回`dummy.next`作为答案; 额外空间复杂度`O(N)`
- 无需创建`dummy`: 就地修改`Oper in place`或者是单纯为了遍历`iterator`计算`tail`或者`len`, 函数返回`Void`; 无额外空间复杂度`O(1)`

二、循环条件怎么写？ `cur.next != null`可以避免空指针，但也会跳过尾节点
- `while (cur != null)        / for ( ; cur != null; cur = cur.next)`
- `while (cur.next != null)   / for ( ; cur.next != null; cur = cur.next)`

## 归并分治
子问题 subproblem, 见HOT100 LC148. 排序链表
```java
    public ListNode sortList(ListNode head) {
        // 如果链表为空或者只有一个节点，无需排序
        if (head == null || head.next == null) {
            return head;
        }
        // 找到中间节点 head2，并断开 head2 与其前一个节点的连接
        // 比如 head=[4,2,1,3]，那么 middleNode 调用结束后 head=[4,2] head2=[1,3]
        ListNode head2 = middleNode(head);
        // 分治
        head = sortList(head);
        head2 = sortList(head2);
        // 合并
        return mergeTwoLists(head, head2);
    }
```
递归最难理解的地方在于, 你能理解「子问题会被排序」，但是看不到这个排序是在哪里完成的。

实际上，排序并不是在 sortList 这一层完成的，而是在不断递归到最小子问题以后，再一层一层 merge 回来完成的。

一递到底
```text
4 -> 2 -> 1 -> 3

左：
4 -> 2
右：
1 -> 3

left = sortList(4->2);
right = sortList(1->3);
return merge(left,right); // 注意：这里不会立刻 merge。因为要先把sortList(4->2)

接下来进入左边
sortList(4->2)

再次二分
4
2

代码又变成
left = sortList(4);
right = sortList(2);
return mergeTwoLists(left,right);

此时4和2都满足
if(head==null || head.next==null)
    return head;

因此sortList(4)直接返回
4

sortList(2)
同样返回
2
```

触底反归
```text
上面终于触底了, 此时上一层代码才能够向下执行mergeTwoLists, 代表归
mergeTwoLists(4,2)整个返回2 -> 4

同理sortList(1->3)整个返回1 -> 3
```

递归的核心思想：当前层不关心子问题是怎么解决的，只相信其依赖的子问题已经按定义完成。
