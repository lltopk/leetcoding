链表中无非是三种指针： `pre`， `cur`， `next`, 需要重点区分两种节点赋值操作：
- 给`p`赋值， 只是移动指针，不会修改链表
- 给`p.next`赋值， 修改节点之间的连接关系，会改变链表结构

带着问题去做下面的题目：

一、两种哨兵节点`dummy`定义方式的区别是什么？是创建新链表还是沿用原链表
- `ListNode dummy = new ListNode()`
- `ListNode dummy = new ListNode(0, head)`

二、循环条件怎么写？ `cur.next != null`可以避免空指针，但也会跳过尾节点
- `while (cur != null)        / for ( ; cur != null; cur = cur.next)`
- `while (cur.next != null)   / for ( ; cur.next != null; cur = cur.next)`

三、构造新的链表作为答案返回，其空间复杂度往往是`O(N)`的，那有没有更优的做法降低空间复杂度为`O(1)`呢？
