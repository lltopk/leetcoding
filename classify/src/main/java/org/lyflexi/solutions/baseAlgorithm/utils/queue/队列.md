## 双端队列ArrayDeque
双端队列可以方便的直接操作队头和队尾
```text
入队: offer(e)->addLast(e)
出队: remove()/pop()->removeFirst()
查看队头: peek()
查看队尾: peekLast()
```
出队之后头部空间会浪费吗? 答案是不会, 这个双端队列是用循环数组实现的
- head: The index of the element at the head of the deque
- tail: The index at which the next element would be added to the tail of the deque, so elements[tail] is always null.
```java
    transient Object[] elements;

    /**
     * The index of the element at the head of the deque (which is the
     * element that would be removed by remove() or pop()); or an
     * arbitrary number 0 <= head < elements.length equal to tail if
     * the deque is empty.
     */
    transient int head;

    /**
     * The index at which the next element would be added to the tail
     * of the deque (via addLast(E), add(E), or push(E));
     * elements[tail] is always null.
     */
    transient int tail;
```
空余的头部还可以给`offer(e)`使用
```java
    public void addLast(E e) {
        if (e == null)
            throw new NullPointerException();
        final Object[] es = elements;
        es[tail] = e;
        if (head == (tail = inc(tail, es.length)))
            grow(1);
    }
```
还有个细节是, 判空逻辑是`head == tail`
```java
    public boolean isEmpty() {
        return head == tail;
    }
```
那么判断满呢? 虽然ArrayDeque没有类似于判断满的API, 为什么? 因为每次在新增元素`es[tail] = e`之后, 如果`head == (tail = inc(tail, es.length))`就直接执行扩容了, 所以本质上这个循环数组不会出现满的情况

上述条件等价于新增元素`es[tail] = e`之后, 如果`es[tail = inc(tail, es.length)] != null`, 表示空位消失了, 违反了约定`elements[tail] is always null.`, 因此等价

至于JDK作者比较头尾指针而不是再次访问数组元素, 是因为性能问题考虑

那有没有办法在新增元素之前就判断满呢? 这也是有些考官喜欢考察的问题答案如下, 这也是用到了`tail`指针永远指向下一个位置这一特性
```java
    public boolean isFull() {
        return ((tail + 1) % capacity) == head;//队列满状态，只剩下1个空位置
    }
```

还有一个意外收获, 与计算机科学比如有符号二进制保持一致的是, 0位是给尾部新增元素用的, 头部新增元素最初只能用-1位, 可以对比下这两个新增方法
- 尾部新增: 先设值, 再增加`tail`索引, `tail`永远指向下一个空位置
- 头部新增: 先减少`head`索引, 后设置, 即`head`永远指向的是真正头部实际位置
```java
    public void addLast(E e) {
        if (e == null)
            throw new NullPointerException();
        final Object[] es = elements;
        es[tail] = e;
        if (head == (tail = inc(tail, es.length)))
            grow(1);
    }

    public void addFirst(E e) {
        if (e == null)
            throw new NullPointerException();
        final Object[] es = elements;
        es[head = dec(head, es.length)] = e;
        if (head == tail)
            grow(1);
    }
```
## 连续出队入队continuous
连续出队入队意思是先出队, 然后将出队的元素再入队. 

由于是双端队列, 涉及到的题目具体有两种策略
- 头部出队 接着 尾部入队
- 尾部出队 接着 头部入队

### 头部出队 接着 尾部入队
见649. Dota2 参议院
```java
/**
 * 双端队列: 头部出队 接着 尾部入队
 */
public class LC649_predictPartyVictory {
    public String predictPartyVictory(String senate) {
        Deque<Integer> radiant = new ArrayDeque<>();
        Deque<Integer> dire = new ArrayDeque<>();
        int n = senate.length();

        // 初始化队列，按顺序存储参议员的索引
        for (int i = 0; i < n; i++) {
            if (senate.charAt(i) == 'R') {
                radiant.offer(i);
            } else {
                dire.offer(i);
            }
        }

        while (!radiant.isEmpty() && !dire.isEmpty()) {
            //无论是行使投票, 或是被对方剔除, 都不能再使用了
            int rIdx = radiant.pop();
            int dIdx = dire.pop();

            // 当前参议员禁止对方阵营的参议员
            if (rIdx < dIdx ) {
                // 胜利了但是还没被投过, 所以下一轮继续投票
                radiant.offer(rIdx + n);//+n表示下轮投票中一定被投
            } else {
                // 胜利了但是还没被投过, 所以下一轮继续投票
                dire.offer(dIdx + n);//+n表示下轮投票中一定被投
            }
        }
        return radiant.isEmpty() ? "Dire" : "Radiant";
    }
}
```

见LC950. 按递增顺序显示卡牌
```java
/**
 * 双端队列
 */
public class LC950_deckRevealedIncreasing {
    /**
     输入: 17,13,11,2,3,5,7
     输出: 2 13 3 11 5 17 7

     先对数组排序，然后从右往左枚举, 用双端队列(先尾部出队, 再加入队头)构造需要的结果。

     排序后，[2,3,5,7,11,13,17]

     17 => 17
     13 => 13 17
     11 => 11 17 13
     7 => 7 13 11 17
     5 => 5 17 7 13 11
     3 => 3 11 5 17 7 13
     2 => 2 13 3 11 5 17 7

     */
    public int[] deckRevealedIncreasing(int[] deck) {

        Arrays.sort(deck);
        Deque<Integer> queue = new ArrayDeque<>();
        for(int i = deck.length - 1; i>=0 ;i--){
            int x = deck[i];
            if(! queue.isEmpty()){
                queue.addFirst(queue.removeLast());
            }
            queue.addFirst(x);
        }

        int[] ret = new int[deck.length];
        for(int i = 0; i< deck.length; i++){
            ret[i] = queue.pop();
        }
        return ret;
    }
}
```

### 先入队 后更新答案
_见LC239. 滑动窗口最大值， 先入队, 再更新答案_

时间复杂度分析: 摊还分析, 虽然是个二重循环, 但最内层`while`循环中的这句代码`q.removeLast();`顶多执行`N`次, 因此时间复杂度是`O(N)`
```java
    //固定长度滑动窗口 + 单调减队列
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        //用双端队列实现单调队列, (单调递减队列)
        Deque<Integer> q = new ArrayDeque<>();
        int[] ans = new int[n - k + 1];
        //单调递减队列
        for(int r = 0; r<n ; r++){
            //0. 当队尾元素与nums[r]相等时, 也移除队尾元素, 这不影响答案, 还能够保证单调队列元素不重复
            while(!q.isEmpty() && nums[q.getLast()] <= nums[r]){
                q.removeLast();
                //单调队列中元素个数可能会小于窗口大小, 这是正常的比如4 2 1遇到了3, 就会剩下4 3
            }

            // 1. 右边入队列, 单调队列中一般存储的是元素下标, 方便下面和窗口左界l0进行判断
            q.addLast(r);
            
            //2. 左边出队列
            int l = r - k + 1;
            //定长滑动窗口每次只会向右移动 1 格, 因此收缩条件这里用if即可
            if(q.getFirst() < l){
                q.removeFirst();
            }

            //3. 当存在左端点时候(>=0), 记录答案
            if(l >=0){
                ans[l] = nums[q.getFirst()];
            }
        }

        return ans;
    }
```

```java
   public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        int[] ans = new int[n - k + 1];
        int[] q = new int[n];
        //q[tail] == null, 即tail永远指向尾部下一个
        int head = 0, tail = 0;
        //维护单调递减的单调队列
        for(int r = 0; r<n ; r++){
            //当队尾元素与nums[r]相等时, 也移除队尾元素, 这不影响答案, 还能够保证单调队列元素不重复
            while(tail != head && nums[q[tail-1]] <= nums[r]){
                tail--;
                //单调队列中元素个数可能会小于窗口大小, 这是正常的比如4 2 1遇到了3, 就会剩下4 3
            }

            // 1. 右边入队列
            //单调队列中一般存储的是元素下标, 方便下面和窗口左界l进行判断
            q[tail++] = r;
    
            //2. 左边出队列

            //定长滑动窗口每次只会向右移动 1 格, 因此收缩条件这里用if即可
            if(q[head] < l){
                head++;
            }
```

下面是个不固定长度的滑动窗口题目, 也是先入队, 后更新答案. 区别是不定长滑动窗口的收缩条件要从`if`改为`while`

见LC1438. 绝对差不超过限制的最长连续子数组, 由于需要比较差值`mx - mn`和`limit`, 所以需要两个单调队列
```java
public class LC1438_longestSubarray {
    //好题目, 不定长滑动窗口 + 单调队列
    //但是用1个队列无法同时保存最大值和最小值,因为单调队列为了维护一种极值，另一种极值的信息会丢失.
    // 比如维护最大值时(递减队列)，较小元素会被循环删除；维护最小值时(递增队列)，较大元素会被循环删除。
    public int longestSubarray(int[] nums, int limit) {
        int ret = 0, l = 0;//滑动窗口求最大长度
        Deque<Integer> dequeDes = new ArrayDeque<>();
        Deque<Integer> dequeInc = new ArrayDeque<>();
        for(int r = 0; r<nums.length; r++){
            while(! dequeDes.isEmpty() && nums[dequeDes.getLast()] < nums[r]){
                dequeDes.removeLast();
            }
            while(! dequeInc.isEmpty() && nums[dequeInc.getLast()] > nums[r]){
                dequeInc.removeLast();
            }

            //1.右边入队列
            dequeDes.offer(r);
            dequeInc.offer(r);

            //2.左边出队列, 不定长滑动窗口收缩的时候要用while
            while(nums[dequeDes.getFirst()] - nums[dequeInc.getFirst()] > limit){
                l++;
                if(l > dequeDes.getFirst()){
                    dequeDes.pop();
                }
                if(l > dequeInc.getFirst()){
                    dequeInc.pop();
                }
            }

            //3.更新答案
            ret = Math.max(ret, r - l + 1);
        }
        return ret;
    }
}
```

## 优先级队列PriorityQueue
本质是JDK由数组实现的堆排序, 对标`PriorityQueue<T>`, 手动实现如下函数
```java
公共函数: 
heapify(...)
offer(...)
poll()

私有函数:
sink(...)
swim(...)
swap(...)
```

给定一个线性数组`nums[]`表示的二叉树, 数组索引为i, 基本算数api如下
```java
    /* 获取左子节点的索引 */
    int left(int i) {
        return 2 * i + 1;
    }
    /* 获取右子节点的索引 */
    int right(int i) {
        return 2 * i + 2;
    }
    /* 获取父节点的索引 */
    int parent(int i) {
        return (i - 1) / 2; // 向下整除
    }
    /* 访问堆顶元素 */
    int peek() {
        return maxHeap.get(0);
    }
```

建堆方式是从最后一个非叶子节点`h.length / 2 - 1`开始逆序枚举节点进行下沉`sink`, 大根堆/小根堆没有本质差别, 以大根堆为例
```java
    // 原地堆化（最大堆）
    // 堆化可以保证 h[0] 是堆顶元素，且 h[i] >= max(h[2*i+1], h[2*i+2])
    private void heapify(int[] h) {
        // 倒着遍历，从而保证 i 的左右子树一定是堆，那么 sink(h, i) 就可以把左右子树合并成一个堆
        // 下标 >= h.length / 2 的元素是二叉树的叶子，无需下沉
        for (int i = h.length / 2 - 1; i >= 0; i--) {
            sink(h, i);
        }
    }
    //从叶节点`h.length / 2`开始没有子节点， 因此它们天然就是合法的子堆，无须堆化, 无须下沉
```
从 Java 代码的实现上看，堆化（heapify）的时间复杂度「似乎」是 O(nlogn) 的？ 但注意堆中大部分节点都集中在下层，下层节点的 sink 函数中循环没几次就结束了

时间复杂度证明过程如下: 假设堆是一棵满二叉树

那么倒数第二层有 4/n 个节点，这些节点在 sink 中至多循环一次。

倒数第三层有 8/n 个节点，这些节点在 sink 中至多循环两次。依此类推，总的循环次数至多为, 总体时间复杂度是`O(N)`
```text
//k == logN
1 * n/(2^2) + 2 * n/(2^3) + ... + (k-1) * n/ (2^k) 
= n * ( 1 * /(2^2) + 2 * /(2^3) + ... + (k-1) * / (2^k) )

设S = 1 * /(2^2) + 2 * /(2^3) + ... + (k-1) * / (2^k)
2S = 1 * /(2) + 2 * /(2^2) + ... + (k-1) * / (2^(k-1))

则2S - S = 1 * /(2) + 1 * /(2^2) + ... + 1 * / (2^(k-1)) - (k-1) * / (2^k)
= 等比数列求和 - (k-1) * / (2^k)
< 1 - (k-1) * / (2^k)
< 1
```
因此总体时间复杂度 < O(N)

完整模板代码如下, 含建堆`heapify(sink)`, 出堆`pop(sink)`, 入堆`offer(swim)`, 见LC1046. 最后一块石头的重量

注意定义`size`变量记录动态长度, 避免循环数组计算长度
```java
class Solution {

    private int size;//用变量记录长度, 避免循环数组计算长度

    public int lastStoneWeight(int[] stones) {
        size = stones.length;

        heapify(stones);

        while (size > 1) {
            int v1 = poll(stones);
            int v2 = poll(stones);

            if (v1 != v2) {
                offer(stones, v1 - v2);
            }
        }

        return size == 0 ? 0 : stones[0];
    }

    // 原地堆化（最大堆）
    // 堆化可以保证 h[0] 是堆顶元素，且 h[i] >= max(h[2*i+1], h[2*i+2])
    private void heapify(int[] h) {
        // 倒着遍历，从而保证 i 的左右子树一定是堆，那么 sink(h, i) 就可以把左右子树合并成一个堆
        // 下标 >= h.length / 2 的元素是二叉树的叶子，无需下沉
        for (int i = size / 2 - 1; i >= 0; i--) {
            sink(h, i);
        }
    }

    // 把 h[i] 不断下沉，每次找左右儿子中最大的交换，直到 i 的左右儿子都 <= h[i] 时停止
    private void sink(int[] h, int i) {
        while (true) {
            int j = 2 * i + 1;

            if (j >= size) {
                break;
            }
            // i 的右儿子比 i 的左儿子大
            if (j + 1 < size && h[j + 1] > h[j]) {
                j++;
            }
            // 说明 i 的左右儿子都 <= h[i]，停止下沉
            if (h[i] >= h[j]) {
                break;
            }

            swap(h, i, j);
            i = j;
        }
    }

    // 上浮, 新的元素从队尾入堆, 需要上浮调整堆结构
    private void swim(int[] h, int i) {
        while (i > 0) {
            //求父节点
            int p = (i - 1) / 2;

            if (h[p] >= h[i]) {
                break;
            }

            swap(h, p, i);
            i = p;
        }
    }

    // 出堆（交换首尾, 删除(size--), 然后下沉）, 记录size--
    private int poll(int[] h) {
        if (size == 0) {
            throw new IndexOutOfBoundsException();
        }

        int ans = h[0];

        swap(h, 0, size - 1);

        size--;

        if (size > 0) {
            sink(h, 0);
        }

        return ans;
    }

    // 入堆(尾部新增, 然后上浮), 记录size++
    private void offer(int[] h, int val) {
        h[size] = val;
        swim(h, size);
        size++;
    }

    // 交换 h[i] 和 h[j]
    private void swap(int[] h, int i, int j) {
        int tmp = h[i];
        h[i] = h[j];
        h[j] = tmp;
    }
}
```

### 数据流逆向定义大小堆
所谓数据流，即是说我们写的算法需要支持 `add()` 函数；在力扣后台评测程序中会多次调用`add()`函数，每次调用都会向我们写的算法中添加一个元素。

而题目要求的类似于在每次 `add()` 之后，整个数据流（包括初始化的元素和所有 add 进来的元素）中的第 K 大元素。

在数据流中, 堆结构扮演的往往是被踢的角色, 如果所求元素(大/小)被踢了, 就覆水难收了, 因此我们要逆向定义大小堆

**然后在调整堆的过程中, 要注意时刻维持堆大小为K, 这样最终堆中就会保留满足题意的K个元素啦, 其中堆顶就是第K大(小顶堆)/K小(大顶堆)的元素了**

如果题目所求第K小, 往往定义大顶堆: 见LC3275. 第 K 近障碍物查询
```java
class Solution {
    public int[] resultsArray(int[][] queries, int k) {
        //大顶堆
        Queue<Integer> priorityQueue = new PriorityQueue<>((a,b) -> (int)(b - a));

        int[] ret = new int[queries.length];
        for(int i = 0; i< queries.length; i++){
            int[] query = queries[i];
            priorityQueue.offer(Math.abs(query[0]) + Math.abs(query[1]));
            if(priorityQueue.size() < k){
                ret[i] = -1;
                continue;
            }
            if(priorityQueue.size() > k){
                priorityQueue.poll();
            }
            ret[i] = priorityQueue.peek();
        }
        return ret;
    }
}
```

```java
        this.k = k;

        while(priorityQueue.size() > k){
            priorityQueue.poll();
        }
        return priorityQueue.peek();
    }
```

### 存储Pair映射关系
堆中同时存储元素的统计维度, 如pair存储元素的索引, 方便映射到原数组

见LC3264. K 次乘运算后的最终数组 I

```java
public class LC3264_getFinalState {
    public int[] getFinalState(int[] nums, int k, int multiplier) {
        Queue<int[]> priorityQueue = new PriorityQueue<>((a, b) -> {
            if (a[0] != b[0]) {
                return a[0] - b[0];
            }
            //当值相等时，PriorityQueue 不保证按下标排序，因此上面要特判
            //题目要求, 找到 nums 中的 最小 值 x ，如果存在多个最小值，选择最 前面 的一个。
            return a[1] - b[1];
        });

        for(int i = 0; i< nums.length; i++){
            priorityQueue.offer(new int[]{nums[i], i});
        }

        while(k-- > 0 && ! priorityQueue.isEmpty()){
            int[] peek = priorityQueue.poll();
            peek[0] *= multiplier;
            nums[peek[1]] = peek[0];
            priorityQueue.offer(peek);
        }

        return nums;
    }
}
```

### 分组(计数)循环
用哈希Map对元素进行计数(相当于分组了), 然后枚举哈希Map, 将枚举值(频次)与优先级队列堆顶(的频次)进行比较

见LC347. 前 K 个高频元素
```java
/**
 * pair优先队列 + 哈希
 */
public class LC347_TopKArray {

    public int[] topKByPriotity(int[] nums, int k) {

        //小顶堆
        Queue<int[]> priorityQueue = new PriorityQueue<>((a, b) -> {
            return a[1] - b[1];
        });
        //分组计数
        Map<Integer, Integer> helper = new HashMap<>();
        for(int x: nums){
            helper.merge(x, 1, Integer::sum);
        }

        int[] ret = new int[k];
        //枚举哈希, 保持优先级队列大小始终为k
        for(int key: helper.keySet()){
            if(priorityQueue.size() == k){
                if(priorityQueue.peek()[1] < helper.get(key)){
                    priorityQueue.poll();
                    priorityQueue.offer(new int[]{key, helper.get(key)});
                }
            }else{
                priorityQueue.offer(new int[]{key, helper.get(key)});
            }
        }

        for (int i = 0; i < k; i++) {
            ret[i] = priorityQueue.poll()[0];
        }
        return ret;
    }
}
```

### 桶排序O(N)
如果是用`Java`的`PriorityQueue`通过循环执行`offer`来建堆, 则最坏情况的时间复杂度是`nlogn`, 那有没有办法进一步优化时间复杂度呢? 

对于一个数组我们思考它的有序性, 很多时候我们关注的只是值的有序性, 却忽略了其实任何一个数组它的索引下标天然就是有序的, 那么我们如何利用索引有序的特性呢?

答案就是桶排序, 我们赋予数组索引特别的业务含义即可, 比如LC347. 前 K 个高频元素

首先计算任意元素出现的最大频次mxCnt, 以此来定义桶数组的长度为mxCnt+1

然后我们将相同出现频次的元素放进相同的桶中, 所以我们应该这样定义桶结构
```java
    List<Integer>[] buckets = new ArrayList[maxCnt+1];
    Arrays.setAll(buckets, idx -> new ArrayList<>());
```

完整代码如下: 
```java
public class LC347_TopKArray2 {
    //面试官? 试试不用优先级队列?
    public int[] topKFrequent(int[] nums, int k) {
        Map<Integer, Integer> cntMap = new HashMap<>();
        int maxCnt = 0;
        for(int x: nums){
            cntMap.merge(x, 1, Integer::sum);
            maxCnt = Math.max(maxCnt, cntMap.get(x));
        }

        //桶排序, 将次数相同的元素归为一桶
        //下标为元素次数, 值为同次数的元素集合
        List<Integer>[] buckets = new ArrayList[maxCnt+1];
        Arrays.setAll(buckets, idx -> new ArrayList<>());
        for(int key: cntMap.keySet()){
            int cnt = cntMap.get(key);
            buckets[cnt].add(key);
        }

        int[] ret = new int[k];
        int j = 0;
        //反向扫描
        for(int i = maxCnt; j < k; i--){
            //竖向扫描
            for(int num: buckets[i]){
                ret[j++] = num;
            }
        }
        return ret;
    }
}
```

### 无限集infinity 
有些特殊的题目并没有给出集合大小, 这被称作无限集问题, 解决这里问题的关键是通过定义指针, 将无限集与优先级队列分开
- 定义`next`指针, 用于无限集的消费, 当优先级队列为空的时候执行`next++`
- 定义优先级队列, 在这类题目中优先级队列存储的是从无限集中访问过的元素

见LC2336. 无限集中的最小数字
```java
public class LC2336_SmallestInfiniteSet {
    //仅存储从无限集中访问过的元素, 且不可重复
    Queue<Integer> priorityQueue;
    int number = 1;
    public LC2336_SmallestInfiniteSet() {
        priorityQueue = new PriorityQueue<>();
    }

    public int popSmallest() {
        //堆为空, 按照无限集的顺序返回
        if(priorityQueue.isEmpty()){
            return number++;
        }
        return priorityQueue.poll();
    }

    public void addBack(int num) {
        if(num < number && !priorityQueue.contains(num)){
            priorityQueue.offer(num);
        }
    }
}
```

### 任务调度「时间轴 + 优先队列」
输入：`tasks = [[1,2],[2,4],[3,2],[4,1]]`, 这类题目属于「时间轴 + 优先队列」
- 排序数组：解决「谁已经到达了」的问题。
- 优先队列：解决「已经到达的任务里先选谁」的问题。

很多题（如会议室、服务器调度、任务调度等）都可以套用下面这个模板：
```java
//按照任务开始时间排序
Arrays.sort(a, (x, y) -> x[0] - y[0]);

while (i < n || !pq.isEmpty()) {

    // 没有可执行任务，直接跳到第一个有效任务的开始时间
    if (pq.isEmpty()) {
        time = Math.max(time, a[i][0]);
    }

    //想象成单线程CPU 让所有已经到达(被阻塞的)的任务入堆
    while (i < n && a[i][0] <= time) {
        pq.offer(a[i]);
        i++;
    }

    // 贪心选择一个任务
    int[] cur = pq.poll();

    // 更新答案
    ...
    // 更新时间
    time += cur[1];
}
```

见LC1834. 单线程 CPU 操作系统经典的短作业优先算法
```java
public class LC1834_getOrder2 {
    public int[] getOrder(int[][] tasks) {
        //把原数组的索引加上, 方便计算过程中映射到答案索引, 也方便给答案排序
        int[][] a = new int[tasks.length][3];
        for(int i = 0; i<tasks.length; i++){
            // int[] task = tasks[i];
            // a[i][0] = task[0];
            // a[i][1] = task[1];
            // a[i][2] = i;
            int[] task = tasks[i];
            a[i] = new int[]{task[0], task[1], i};
        }
        //按照任务开始时间排序
        Arrays.sort(a, (x, y) -> x[0] - y[0]);
        //按照任务耗时和索引, 建立小顶堆
        Queue<int[]> pq = new PriorityQueue<>((x, y) -> {
            if(x[1] != y[1]){
                return x[1] - y[1];
            }
            return x[2] - y[2];
        });
        int[] ret = new int[tasks.length];
        int time = 0, k = 0, n = tasks.length;
        for(int i = 0; i < n || !pq.isEmpty(); ){
            if(pq.isEmpty()){
                //已经按照任务开始时间排序了, 所以直接跳到第一个任务开始时间
                time = Math.max(time, a[i][0]);
            }
            //单线程CPU, 当前时间time可能阻塞了很多后续的任务, 这些阻塞的任务通通入堆
            while(i< n && a[i][0] <= time){
                pq.offer(a[i++]);
            }
            int[] peek = pq.poll();
            time += peek[1];
            ret[k++] = peek[2];
        }
        return ret;
    }
}
```

### 给定N个单调序列
这种题目的前提必须是N个单调序列, 递增递减都可以, 这是核心特征, 有两种考法
- 按照总体递增/递减顺序返回, 只需要题目给N个序列即可
- 计算总体K次的最大收益和, 这种需要题目明确给出K值

给你N个单调递增链表序列, 按照总体递增顺序返回。见LC23. 合并 K 个升序链表
```java
/**
 * K个单调序列  +  优先级队列
 */
public class LC23_mergeKLists {
    public ListNode mergeKLists(ListNode[] lists) {
        PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);
        for (ListNode head : lists) {
            if (head != null) {
                pq.offer(head); // 把所有非空链表的头节点入堆
            }
        }
        ListNode dummy = new ListNode();
        ListNode cur = dummy;
        while(!pq.isEmpty()){
            ListNode peek = pq.poll();
            if(peek.next != null){
                pq.offer(peek.next);
            }
            cur.next = peek;
            cur = cur.next;
        }
        return dummy.next;
    }
}
```

还有些题目特征不太明显, 你需要点数学功底, 见1792. 最大平均通过率, 给你N个单调递减序列, 操作K次, 保证总体收益最大

```java
/**
 * K个单调序列  +  优先级队列
 */
public class LC1792_maxAverageRatio {
    //我们关注的是通过率增量
    private double gain(int p, int t) {
        return (double) (p + 1) / (t + 1) - (double) p / t;
    }
    //事实上, 随着增加的学生越来越多，通过率的增量是递减（非递增）的
    //这相当于给了我们n个单调递减序列, 想象成一个二维数组int[N][], 所有班级的通过率之和的最大值相当于： 给你 n 个递减（非递增）序列，第一项必选（链表头, 初始通过率），然后再从剩余N个链表中选出最大的 extraStudents 个数（增量），计算这 n+extraStudents 数的元素和。
    // 在计算层面我们只是依据增长率做贪心, 但是无需具体求解出增长率, 我们可以先累加增长量, 最后统一计算平均增长率
    public double maxAverageRatio(int[][] classes, int extraStudents) {
        PriorityQueue<int[]> pq = new PriorityQueue<>(
                (a, b) -> Double.compare(
                        gain(b[0], b[1]),
                        gain(a[0], a[1])
                )
        );
        for(int[] cls: classes){
            pq.offer(cls);
        }

        while (extraStudents-- > 0) {
            int[] cur = pq.poll();
            //关键, 每次加入1个万能学生之后, 分子分母都得加1
            cur[0]++;
            cur[1]++;
            //此时的cur相当于next, 将next入堆, 因为全堆排序是动态调整的
            pq.offer(cur);
        }
        double ret = 0;
        while (!pq.isEmpty()) {
            int[] c = pq.poll();
            ret += (double) c[0] / c[1];
        }
        return ret / classes.length;
    }
}
```
