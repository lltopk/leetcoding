如果让你计算 `1234×6789` 的个位数，你会如何计算？ 由于只有个位数会影响到乘积的个位数，那么 `4×9=36` 的个位数 `6` 就是答案。

对于 `1234+6789` 的个位数，同理，`4+9=13` 的个位数 `3` 就是答案。

取模结果一定在`[0, mod-1]`, 这个结论同样适用于二进制计算

## 加乘法取模
根据带余除法定理, 任意整数 `a` 都可以表示为 `a=qm+r (m!=0)`， 其中整数 `q` 为 `a` 除以 `m` 的商（quotient），整数 `r` 为 `a` 除以 `m` 的余数（remainder），即 `r = a mod m`

设整数 `a = q1*m + r1`, `b = q2*m + r2` 。可证明
```text
//启发: 可用于防止溢出问题
(a+b) % m = ((a % m) + (b % m)) % m
//启发: 可用于防止溢出问题
(a*b) % m = ((a % m) * (b % m)) % m
```

## 负数取模
当对负整数求模, 如`-1 % 4` 结果是`-1`, 这是错误的, 正确结果应该是`3`, 那么如何让上述公式兼容负数的情况呢?

根据同余定理, 两个整数 `x` 和 `y`，如果 `(x−y) mod m = 0`（也就是 `x−y` 是 `m` 的倍数），则称 `x` 与 `y` 关于模 `m` 同余，记作`x 三 y (mod m)`

符合同余定理的两个数对目标取模余数相等, 因此只要将负数映射到它的同余镜像正数即可

以`-1`和`3`为例, 根据同余公式`-1 三 3 (mod 4)`, 我们可以通过`-1 % 4 + 4` 可得 镜面`3`

一般的, 镜像`y = x % m + m`, 因此在代码实现时，如果被模数存在负数的可能, 需要先将被模数`x`转化为 `x % m + m` 再对 m 求模

**特别的, 当`|x| < m`的时候, `y = x + m` 即可, 本质是保证`x + m`落在`[0, m-1]`即可**

## 减法取模
同样根据带余除法定理, 可推导
```text
//启发: 可用于两数之和问题
(a-b) % m = ((a % m) - (b % m)) % m
```

该公式可用于解题, 但是不够严谨, 原因是存在负数`- (b % m)`, 

**所以在具体的代码执行层面, 我们需要将`- (b % m)`转为正数** 

由于`|- (b % m)| < m`, 因此只需要`+m`便能保证`- b % m + m`的结果落在`[0, m-1]`
```text
(a-b) % m = ((a % m) - b % m + m) % m
```

## 除法取模

//推导过程需要用到二项式定义与费马小定理: 见: https://leetcode.cn/discuss/post/3584387/fen-xiang-gun-mo-yun-suan-de-shi-jie-dan-7xgu/

## 快速幂
如何计算`x`的`n`次幂`pow(x,n)`, 快速幂推导如下

将n用二进制表示, 例如求`pow(x, 13)`, `13`表示为二进制是`1101`

计算过程为, 从低到高枚举二进制的1位, 并以该位代表的实际二进制值作为幂, 累乘即x * x^4 * x^8, 见LC50.Pow(x, n)
```java
    public double myPow(double x, int N) {
        double ans = 1;
        //防止输入n= -2147483648, 导致正2147483648溢出
        long n = N;
        //x^(-n) = (1/x)^n
        if(n<0){
            x = 1/x;
            n = -n;
        }

        //快速幂, 将n用二进制表示, 例如求pow(x, 13), 13表示为二进制是1101
        //计算过程为, 从低到高枚举二进制的1位, 并以该位代表的实际二进制值作为幂, 累乘即x * x^4 * x^8
        while(n!=0){
            if((n & 1) == 1){
                ans *= x; 
            }
            //以任意位代表的实际二进制值作为幂, 如1, 2, 4, 8
            x *= x;
            n >>= 1;
        }
        return ans;
    }
```

本来快速幂和取模操作没有关系, 但是当题目`x`不是`double`类型的时候, 需要引入取模运算来解决数值溢出问题
```java
    /**
     * N <= 10^9
     * @param x long型, 避免乘法x* x % mod;在取模前溢出
     * @param N
     * @param mod
     * @return
     */
    public int myPow(long x, int N, int mod) {
        //long, 避免乘法ans * x % mod;在取模前溢出
        long ans = 1;
        int n = N;
        if(n<0){
            x = 1/x;
            n = -n;
        }
        
        while(n!=0){
            if((n & 1) == 1){
                //必须先乘成后取模, 不可写成ans *= x%mod;
                ans = ans * x % mod; 
            }

            //必须先乘后取模, 不可写成x *= x%mod;
            x = x* x % mod;
            n >>= 1;
        }
        return (int)ans;
    }
```
