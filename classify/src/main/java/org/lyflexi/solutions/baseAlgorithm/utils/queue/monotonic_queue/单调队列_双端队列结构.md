个人觉得叫单调双端队列更准确。

单调队列 = 滑动窗口 + 单调栈。必须先掌握滑动窗口和单调栈这两个知识点，再学单调队列。

入队、出队、更新答案，这三步的顺序如何思考？ 有两种情况。 
1. 如果更新答案时，用到的数据包含当前元素，那么就需要先入队，再更新答案；
2. 如果更新答案时, 用到的数据不包含当前元素，那么就需要先更新答案，再入队。
3. 至于出队，一般写在前面, 因为要及时去掉无用数据`queue.removeLast()/queue.removeLast()`, 保证双端队列有序, 且队列元素均在窗口内

## 先入队, 再更新答案
见LC239. 滑动窗口最大值
```java
    //固定长度滑动窗口 + 优先级队列(堆)
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        //用双端队列实现单调队列, (单调递减队列)
        Deque<Integer> q = new ArrayDeque<>();
        int l = 0;
        int[] ans = new int[n - k + 1];
        for(int r = 0; r<n ; r++){
            // 1. 右边入
            //当队尾元素与nums[r]相等时, 也移除队尾元素, 这不影响答案, 还能够保证单调队列元素不重复
            while(!q.isEmpty() && nums[q.getLast()] <= nums[r]){
                q.removeLast();
                //单调队列中元素个数可能会小于窗口大小, 这是正常的比如4 2 1遇到了3, 就会剩下4 3
            }

            //单调队列中一般存储的是元素下标, 方便下面和窗口左界l0进行判断
            q.addLast(r);


            //2. 左边出
            int l0 = r - k + 1;
            //由于每次入队最多1次, 因此出队也至多只有1次, 所以下面用if即可
            if(q.getFirst() < l0){
                q.removeFirst();
            }

            //3. 当存在左端点时候(>=0), 记录答案
            if(l0 >=0){
                ans[l = l0] = nums[q.getFirst()];
            }
        }

        return ans;
    }
```
