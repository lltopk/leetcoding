对于原数组 `a`，定义其差分数组`（difference array）`为
```text
d[i]=a[0], i==0
d[i]=a[i] − a[i−1], i≥1
```
有差分数组 <-> 原数组 <-> 前缀和数组, 这类似于导函数<->原函数<->积分的关系, 因此求差分、前缀和是一对互逆的过程
- 差分->原数组：对差分求前缀和
- 原数组->前缀和：对原数组求前缀和
- 前缀和->原数组：对前缀和求差分
- 原数组->差分：对原数组求差分

注意, 没有要求差分数组与原数组的长度相等, 你首先需要做的是确定差分数组的长度, 差分数组有如下两个性质:
```text
性质 1：从左到右累加 `d` 中的元素，可以得到原数组 `a` 中此刻的值, 即原数组中一个点的值代表差分数组求和到此。

性质 2: 在差分数组的性质 2 中, 如下两个操作是等价的
- 把覆盖次数矩阵 `a` 的子数组 `a[i],a[i+1],…,a[j]` 都加上 `x`。
- 等价于(一人得道鸡犬升天)把差分值 `d[i]` 增加 `x`， 但是不能影响原有的`a[j+1]`, 因此要把差分值 `d[j+1]` 扣除 `x`。
利用性质 2，我们仅需要通过修改两个差分值`d[i]` 和 `d[j+1]`, 即可映射覆盖次数矩阵的批量修改, 这只需要 `O(1)` 的时间。 最后再利用性质 1 从差分数组复原覆盖次数矩阵 a。
```

实际上差分数组还原出的原数组和题目给出的初始矩阵并不是一个东西, 

**"原数组"更多是个计算过程中的业务表述, 特点都是为以横轴刻度为索引, 不同点在于你需要根据具体的题目, 将什么定义为"原数组"值**
- 覆盖次数矩阵
- 实际数值应用
- 应当减去次数

## 覆盖次数矩阵
见LC2848. 与车相交的点, 这道题目设原数组为, 任意横轴刻度, 该刻度被任意线段覆盖的次数
```java
    public int numberOfPoints(List<List<Integer>> nums) {
        //1. 确定差分数组大小
        int maxEnd = 0;
        for(List<Integer> interval: nums){
            maxEnd = Math.max(maxEnd, interval.get(1));
        }
        int len = maxEnd+1;
        //下面要用到差分数组性质2, 唯二差分值中第二个值是j+1, 为了避免越界特判, 因此这里长度加1
        int[] diff = new int[len+1];

        //2. 利用性质2计算差分数组
        for(List<Integer> interval: nums){
            diff[interval.get(0)]++;
            diff[interval.get(1) + 1]--;
        }


        //3. 利用性质1还原覆盖次数数组
        int s = 0, ans = 0;
        for(int x: diff){
            s+=x;
            if(s>0){
                ans++;
            }
        }
        return ans;
    }
```
补充说明的是, 多出来的差分索引这个只是尾部哨兵位, 最终在还原覆盖数组的时候有效计算依然是`[0, len)`, 不对结果造成影响.

## 实际数值应用
下面再看一道题, 见LC1094. 拼车 , 这道题目设"原数组"为横轴任意点位时刻情况下, 车内乘客数量
_但是因为这道题to位置并不包含乘客计数, 因此无需补充尾部哨兵位, 自然就无需让差分数组长度+1了_
```java
    //差分数组
    public boolean carPooling(int[][] trips, int capacity) {
        //1. 确定差分数组的长度, 题目中给定的0 <= fromi < toi <= 1000
        int N = 1001;
        int[] diff = new int[N];//由于乘客计数不包括to(已经下车), 因此性质2不存在边界条件, 所以差分数组长度不用再+1了
        
        //2. 利用性质2计算差分数组
        //差分数组只关注from和to点位, 其他点位无需计算因为车内状态没有变化
        // 车上人trip[0], 意味着从[from, to-1]都多trip[0]人
        // 车下人trip[0], 意味着从[to]开始车内减trip[0]人
        for(int[] trip: trips){
            int x = trip[0], from = trip[1], to = trip[2];
            diff[from] += x;
            diff[to] -= x;//这里其实是j=to-1, j+1=to
        }
        
        //3. 差分数组求和, 利用性质1还原原数组
        int s = 0;//空间优化, 不需要完全存储原数组
        for(int x: diff){
            s+=x;
            if(s > capacity){
                return false;
            }
        }

        return true;
    }
```

## 应当减去次数
见LC3355 零数组变换 I

这类题目, 差分数组求和得到的是当前位置`i`, 对于原数组值`nums[i]`来说应当减去的次数
```java
    public boolean isZeroArray(int[] nums, int[][] queries) {
        int n = nums.length;
        //1. 差分数组要用到性质2, 因此长度+1
        int[] diff = new int[n+1];
    
        //2. 计算差分数组
        for(int[] query: queries){
            diff[query[0]]++;
            diff[query[1]+1]--;
        }
    
        //3. 对差分数组求和, 代表的是该位置对于nums[i]来说应当减去的次数
        int s = 0;
        for(int i = 0; i<n; i++){
            s += diff[i];
            if(s < nums[i]){//nums[i] - s无法到0
                return false;
            }
        }
    
        return true;
    }
```
## 最小代价和/最小操作次数
见LC3914 使数组非递减需要的最小累计值
```java
    public long minOperations(int[] nums) {
        long ans = 0;
        //diff[0]==nums[0] > 0, 无需代价, 所以第一项不用管,
        for(int i = 1; i<nums.length; i++){
            if(nums[i] - nums[i-1] < 0){
                //我们只考虑付出的代价Math.abs(diff[i]), 不考虑diff[j+1], 这样就最小化了答案
                //这样我们每次都是对尾子数组[i, n)操作, 忽略diff[n]--
                ans += Math.abs(nums[i] - nums[i-1]);
                
                //即使后面存在比nums[i]更小的数, 因为是同时+1, 所以无法抵消已经付出的代价Math.abs(diff[i]), 因此不会漏算
            }
        }
        return ans;
    }
```
## 双数组求差分
这类题目先计算双数组本身的差值数组, 然后对差值数组求差分数组, 见LC3229 使数组等于目标数组所需的最少操作次数
```java
    public long minimumOperations(int[] nums, int[] target) {
        //最小操作次数 = min(posSum - negSum) + |posSum - negSum|, 前者是同时改变d[i]和d[j+1]的情况, 后者是只改变d[i]的情况
        //上式子等价于max(posSum, negSum)
        long posSum = 0, negSum = 0;
        for(int i = 0; i<nums.length; i++){
            int d = 0;
            if(i==0){
                d = nums[i] - target[i];
            }else{
                d = (nums[i] - target[i]) - (nums[i-1] - target[i-1]);
            }

            if(d>0){
                posSum += d;
            }else{
                negSum += Math.abs(d);
            }
        }
        return Math.max(posSum, negSum);
    }
```

## 二维差分数组
见LC2132. 用邮票贴满网格图 两个关键点
1. 如何判断邮票可放, 这用二维前缀和为0判断, 如果可放, 批量更新覆盖次数矩阵visit[][]中与被贴住范围对应的格子
2. 如何标记格子被放, 这用二维差分性质2快速还原覆盖次数矩阵中每个格子+1操作, 这可以省略覆盖次数矩阵visit[][]

最后通过二维差分数组还原覆盖次数矩阵, 如果还原过程中存在差分子和为0, 则返回false

最终还原覆盖次数矩阵的过程中, 可以借鉴一维差分还原原数组那样做空间优化, 不必还原出整个覆盖次数矩阵
```java
    //二维差分题目, 还原覆盖次数矩阵的时候空间优化
    public boolean possibleToStamp(int[][] grid, int stampHeight, int stampWidth) {
        //两个关键点
        //1. 如何判断邮票可放, 这用二维前缀和为0判断, 如果可放, 批量更新覆盖次数矩阵visit[][]中与被贴住范围对应的格子
        //2. 如何标记格子被放, 这用二维差分性质2快速还原覆盖次数矩阵中每个格子+1操作, 这可以省略覆盖次数矩阵visit[][]
        //最后通过二维差分数组还原覆盖次数矩阵, 如果还原过程中存在差分子和为0, 则返回false

        int m = grid.length, n = grid[0].length;
        int[][] preS = new int[m+1][n+1];
        for(int i = 0; i<m; i++){
            for(int j = 0; j<n; j++){
                preS[i+1][j+1] = preS[i][j+1] + preS[i+1][j] - preS[i][j] + grid[i][j];
            }
        }

        //定义差分数组长度, 因为要用到性质2, 所以长度都要加1
        int[][] diff = new int[m+1][n+1];
        for(int i = stampHeight-1; i<m; i++){//枚举邮票右下角
            for(int j = stampWidth-1; j<n; j++){
                int i0 = i - stampHeight + 1;
                int j0 = j - stampWidth + 1;
                int subS = preS[i+1][j+1] - preS[i+1][j0] - preS[i0][j+1] + preS[i0][j0];
                if(subS == 0){
                    //维护差分数组的四个顶点
                    diff[i0][j0]++;
                    diff[i0][j+1]--;
                    diff[i+1][j0]--;
                    diff[i+1][j+1]++;
                }
            }
        }

        //二维差分数组求和空间优化
        //以右下角(>0, >0)作为顶点的前缀和, 都依赖左 上 左上, 因此二维差分数组也可以就地求和
        //那么第一列和第一行怎么办? 当作一维数组处理
        for(int i = 0; i<m; i++){
            for(int j = 0; j<n; j++){
                // 在二维差分求和过程中, 我们可以用差分和替换差分值, 这样就可以利用历史求和数据计算新的差分和, 这不影响新的差分和计算
                //此时 diff 已经不再是差分数组, 而是恢复后的覆盖次数矩阵
                if(i > 0) diff[i][j] += diff[i-1][j];

                if(j > 0) diff[i][j] += diff[i][j-1];

                if(i > 0 && j > 0)
                    diff[i][j] -= diff[i-1][j-1];
                if(diff[i][j] == 0 && grid[i][j] == 0){
                    return false;
                }
            }
        }
        return true;
    }
```
