红黑树有时候很有用, 其键的集合`keys`是有序的, 实现了二分查询目标元素的效果, 时间复杂度是`logn`

在Java中有两个集合都用到了红黑树
- TreeSet: 适用于不需要计数的场景, 比如迭代求最大/最小
- TreeMap: 适用于计数场景, 统计总量/总数

其中`TreeSet`持有`TreeMap`, `TreeMap`持有内部类`Entry`， 不同于`HashMap`, 这里的`Entry`是红黑树节点
```java
    // Red-black mechanics

    private static final boolean RED   = false;
    private static final boolean BLACK = true;

    /**
     * Node in the Tree.  Doubles as a means to pass key-value pairs back to
     * user (see Map.Entry).
     */

    static final class Entry<K,V> implements Map.Entry<K,V> {
        K key;
        V value;
        Entry<K, V> left;
        Entry<K, V> right;
        Entry<K, V> parent;
        boolean color = BLACK;

        /**
         * Make a new cell with given key, value, and parent, and with
         * {@code null} child links, and BLACK color.
         */
        Entry(K key, V value, Entry<K, V> parent) {
            this.key = key;
            this.value = value;
            this.parent = parent;
        }
    }
```

## TreeSet
快速查找, 适用于不需要计数的场景, 比如迭代求最大/最小
```java
TreeSet<Integer> treeSet = new TreeSet<>();

//获取比key小的key, 严格小于
treeSet.lower(e);
//获取比key小的key, 小于等于
treeSet.floor(e);
//获取比key大的key, 严格大于
treeSet.higher(e);
//获取比key大的key, 大于等于
treeSet.ceiling(e);
//获取最后一个key
treeSet.last(e);
//获取第一个key
treeSet.first(e);
```

## TreeMap
和`TreeSet`的`api`类似, 由于是`map`, 多了获取`entry`的`api`
```java
TreeMap<Integer, Integer> treeMap = new TreeMap<>();

//获取比key小的key, 严格小于
treeMap.lowerKey(key);
//获取比key大的key, 严格大于
treeMap.higherKey(key);
//获取比key小的entry, 严格小于
treeMap.lowerEntry(key);
//获取比key大的entry, 严格大于
treeMap.higherEntry(key);

//获取比key小的key, 小于等于
treeMap.floorKey(key);
//获取比key大的key, 大于等于
treeMap.ceilingKey(key);
//获取比key小的entry, 小于等于
treeMap.floorEntry(key);
//获取比key大的entry, 大于等于
treeMap.ceilingEntry(key);


//获取最后一个key
treeMap.lastKey()
//获取第一个key
treeMap.firstKey()
```

## SortedList
`python`自带了有序集合(排序集合)`SortedList`，(允许重复元素), 往往用来计算`rank`。底层数据结构是分块数组，块内有序，并且块间整体有序
- 查找是两轮二分，第一轮确定块，第二轮查找块内位置，复杂度是两个`log(n)`
- 插入也是两轮二分，找到块内位置之后开始移动块内元素，复杂度是`log(n) + O(1000)`，约等于`log(n)`

每个块默认长度是`1000`，超过`1000`会进行分裂

但很遗憾`Java`没有这样现成的有序集合, 只有`ArrayList`, `ArrayList`只是能够根据数组索引访问, 但元素值是无序的，因此本质上依然是无序的

即使是红黑树`TreeMap`或者`TreeSet`的有序性也只是体现在中序遍历上, 因此红黑树只知道目标节点的左右大小关系, 无法对整体数组进行`rank`
